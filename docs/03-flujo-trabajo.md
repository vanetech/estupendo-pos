# Flujo de Trabajo con Git

## Inicialización
- `git init` → crear repo local
- `git remote add origin git@github.com:vanetech/estupendo-pos.git` → conectar con GitHub

## Commits
- `git add .` → agregar cambios
- `git commit -m "mensaje"` → guardar cambios localmente
- `git push` → subir cambios a GitHub

## Buenas prácticas
- Commits pequeños y claros
- Documentar decisiones importantes en `docs`
- Una rama principal (`main`) por ahora
